## 0. 왜 이 문서가 필요한가 — 문제 정의

XBRL로 "매출액 100억"을 태깅하려면 **"언제의 매출액인가"(기간)**가 반드시 있어야 한다.
그런데 KIND에서 내려받은 잠정실적 공시의 기간 표기는 **기업마다 완전히 제각각**이다:

```
(25년2분기)   2024.2Q     2Q24       4Q'23       FY'24.2분기
'25.2Q        2025 2분기   1Q `25     (2025.8월)   제71기 2분기
(2025.4.1~2025.6.30)      2023년 4분기(2023.10.01~2023.12.31)
(-)  ( )  16314344(비정상)   …그리고 라벨이 아예 없는 경우
```

같은 "2024년 2분기"를 **여덟 가지 넘게** 다르게 쓴다. 게다가:
- **연도를 안 쓰는** 회사가 많다 (`2분기`만 씀 → 몇 년도?)
- **분기가 연도 앞**에 오기도(`2Q24`), **뒤**에 오기도(`24.2Q`) 한다
- **기수**(제71기)를 쓰면 그건 연도가 아니다
- 일부 서식은 날짜칸에 **엉뚱한 숫자**(재무값)가 들어가 있다
- **월 단위**로 내는 회사(현대차 판매대수, 파라다이스 카지노매출)도 있다

→ **정규식 하나로는 절대 못 잡는다.** 그래서 "**여러 단서를 결합**"하는 표준화 엔진이 필요하다.

## 1. 핵심 개념과 용어

| 용어 | 뜻 |
|---|---|
| **당기(current)** | 이번에 발표하는 실적의 기간 (예: 2025년 2분기) |
| **전기(previous)** | 직전 분기/월 (예: 2025년 1분기) |
| **전년동기(yoy)** | 1년 전 같은 기간 (예: 2024년 2분기) |
| **기수정보** | 공시서식에서 각 기간을 적는 칸의 줄글 표기 |
| **라벨(label)** | 기수정보 칸에 실제로 쓰인 문자열 (`(25년2분기)` 등) |
| **카덴스(cadence)** | 그 회사가 분기로 내는지 월로 내는지 |
| **제출시점 휴리스틱** | 언제 냈는지(제출월)로 어느 분기인지 추정하는 규칙 |
| **신뢰도(confidence)** | 확정 근거의 강도 (high / med / low) |

**설계 대원칙 두 가지**

1. **당기(current)만 견고하게 확정**한다. → 전기·전년동기는 당기에서 **계산으로 파생**.
   (회사가 전기/전년동기 라벨을 틀리게 써도, 당기만 맞으면 자동 교정된다.)
2. **여러 단서를 우선순위로 결합**한다. 명시적 날짜 > 라벨 > 제출시점 > 시계열.

## 4. 정규표현식 5분 입문 (컴알못용)

정규표현식(regex)은 **"글자 패턴을 찾는 규칙"**이다. 아래 기호만 알면 이 문서를 읽을 수 있다.

| 기호 | 뜻 | 예 |
|---|---|---|
| `\d` | 숫자 한 글자 (0~9) | `\d\d` → "25" |
| `\D` | 숫자가 **아닌** 글자 한 개 | `.`, `년`, 공백 |
| `\s` | 공백류(스페이스·탭) | " " |
| `{2,4}` | 앞의 것이 **2~4개** | `\d{2,4}` → "25" 또는 "2025" |
| `?` | 앞의 것이 **0개 또는 1개** (있어도 없어도 됨) | `'?` → 작은따옴표 있거나 없거나 |
| `*` | 앞의 것이 **0개 이상** (없거나 여러 개) | `\s*` → 공백 없거나 여러 개 |
| `[…]` | 괄호 안 글자 중 **아무거나 하나** | `[년.\s]` → '년' 또는 '.' 또는 공백 |
| `[1-4]` | 1,2,3,4 중 하나 | 분기 숫자 |
| `(…)` | **그룹** — 나중에 값을 꺼내 씀 | `(\d{4})` → 연도만 추출 |
| `(?:…)` | 그룹이되 **꺼내지 않음** (묶기만) | |
| `|` | **또는** | `분기|Q` → '분기' 또는 'Q' |
| `\b` | **단어 경계** (숫자·글자의 끝) | 뒤에 딴 숫자 안 붙게 |

두 가지 사용 함수:
- `re.search(패턴, 글자)` : 글자 **아무 위치**에서 패턴을 찾는다.
- `re.I` : **대소문자 무시** (q 든 Q 든 같게).

> 읽는 법: 정규식을 왼쪽→오른쪽으로 "이런 글자 다음, 이런 글자…"로 소리 내 읽으면 된다.
> 아래 §6.2에서 실제 패턴을 **토큰 하나씩** 풀어 설명한다.

## 5. 축 ① — v2026 명시적 날짜

2026년 신서식(v5.11/5.09)은 기간을 **줄글이 아니라 시작일/종료일 날짜칸**으로 준다.
가장 신뢰도가 높으므로 **최우선**으로 쓴다. 단, **날짜가 상식적일 때만**.

```
period_dates = { current:[시작,종료], previous:[…], yoy:[…] }
        │
        ▼  _valid_range() 로 세 기간 모두 검사
   ┌────────────────────────────────────────────┐
   │ 연도 2015~2035?  월 1~12?  일 1~31?  시작≤종료? │
   └────────────────────────────────────────────┘
        │ 통과                        │ 실패 (예: 16314344 → 1631-43-44)
        ▼                            ▼
   그대로 채택 [high]          축 ②·③으로 폴백
```

**왜 유효성 검사가 필요한가?** 금융지주·증권 등 일부 회사는 이 날짜칸에 **재무 숫자**가
잘못 들어가 있어(`16314344` 같은 8자리) 그대로 쓰면 "1631년 43월 44일" 같은 쓰레기가 된다.
→ 검사에서 걸러 제출시점 휴리스틱(축 ③)으로 넘긴다.

```python
def _valid_date(s):   # 'YYYY-MM-DD'가 상식적 범위인지
    m = re.fullmatch(r"(\d{4})-(\d{2})-(\d{2})", s)   # 정확히 4자리-2자리-2자리
    y, mo, da = map(int, m.groups())
    return 2015 <= y <= 2035 and 1 <= mo <= 12 and 1 <= da <= 31
```
- `re.fullmatch` : 문자열 **전체**가 패턴과 딱 맞아야 함(일부만 맞으면 탈락).
- `(\d{4})-(\d{2})-(\d{2})` : 숫자4개 · `-` · 숫자2개 · `-` · 숫자2개 → 연/월/일 세 그룹.

## 6. 축 ② — 라벨(줄글) 파싱

`parse_label()` 이 담당. 아래 순서로 시도하고, 먼저 맞는 형식을 채택한다.

```
parse_label("(25년2분기)")
   ├─ 1. 명시적 범위?   2025.4.1~2025.6.30 형태            → range
   ├─ 2. 월?           2025.8월 / '25년 8월  (분기표기 없을때) → month
   ├─ 3. 반기?         상반기 / H1                          → half
   ├─ 4. 분기(b)?      2Q24 (분기 먼저, 연도 뒤)            → quarter
   ├─ 5. 분기(a)?      24년2분기 (연도 먼저, 분기 뒤)        → quarter
   └─ 6. 다 실패        → None (축 ③ 제출시점으로)
```

### 6.1 라벨 형식 카탈로그

| 실제 표기 예 | 분류 | 뽑아내는 값 |
|---|---|---|
| `(2025.4.1~2025.6.30)` | range | 시작 2025-04-01, 종료 2025-06-30 |
| `(2025.8월)` `'25년 8월` | month | 연 2025, 월 8 |
| `(25년 상반기)` `2025 H1` | half | 연 2025, 반기 1 |
| `2Q24` `4Q'23` `2Q 2024` `1Q \`25` | quarter(b) | 분기·연도 (연도가 뒤) |
| `24년2분기` `2024.2Q` `'24.2Q` `FY'24.2분기` | quarter(a) | 분기·연도 (연도가 앞) |
| `제71기 2분기` | quarter(a) | 분기 2, **연도=없음**(기수 무시) |
| `(-)` `( )` | — | None (라벨 없음) |

### 6.2 형식별 정규식 + 한 줄씩 해설

#### (1) 명시적 범위 — `2025.4.1 ~ 2025.6.30`

```
(\d{4})\D{1,2}(\d{1,2})\D{1,2}(\d{1,2})\s*[~\-]\s*(\d{4})\D{1,2}(\d{1,2})\D{1,2}(\d{1,2})
```
| 조각 | 읽는 법 |
|---|---|
| `(\d{4})` | 연도 4자리 (2025) |
| `\D{1,2}` | 숫자 아닌 구분자 1~2개 (`.` 또는 `년 ` 등) |
| `(\d{1,2})` | 월 1~2자리 |
| `\D{1,2}(\d{1,2})` | 구분자 + 일 |
| `\s*[~\-]\s*` | (공백) **물결 또는 하이픈** (공백) = "~" 구분 |
| 뒤 절반 | 종료일도 같은 방식 |

→ 여섯 그룹(연·월·일 × 2)을 뽑아 `2025-04-01`, `2025-06-30`으로 조립.
숫자가 상식 범위(연 2015~2035, 월 1~12, 일 1~31)일 때만 채택(쓰레기 범위 방지).

#### (2) 월 — `2025.8월`, `'25년 8월`

```
(?:(\d{2,4})\s*[년.\s]*)?(\d{1,2})\s*월
```
| 조각 | 읽는 법 |
|---|---|
| `(?: … )?` | **통째로 있어도 없어도 됨** (연도 생략 대비) |
| `(\d{2,4})` | 연도 2~4자리 (25 또는 2025) |
| `\s*[년.\s]*` | 공백/`년`/`.` 이 여러 개 |
| `(\d{1,2})` | 월 |
| `\s*월` | (공백) '월' 글자 |

→ 단, **분기 표기가 있으면 이 규칙은 건너뛴다**(`has_q` 검사). "2025.8월"에는 분기가 없으니 월로 인식.

#### (3) 분기 (b) — 분기가 **먼저**, 연도가 **뒤**: `2Q24`, `4Q'23`, `1Q \`25`

```
([1-4])\s*(?:분기|Q)\s*[.'`\s]*(\d{2,4})\b
```
| 조각 | 읽는 법 |
|---|---|
| `([1-4])` | 분기 숫자 (1~4만) |
| `\s*(?:분기|Q)` | (공백) '분기' **또는** 'Q' |
| `\s*[.'`\s]*` | 뒤따르는 **점·작은따옴표·백틱·공백** 여러 개 (`'`, `` ` ``, `.` 대응) |
| `(\d{2,4})` | 연도 |
| `\b` | 단어 경계 — 뒤에 엉뚱한 숫자가 안 붙게 |

→ 이 배열은 **연도가 뒤에 명시**돼 있으므로 가장 확실 → **먼저** 시도한다.

#### (4) 분기 (a) — 연도가 **먼저**, 분기가 **뒤**: `24년2분기`, `2024.2Q`, `FY'24.2분기`

```
(?:'?(\d{2,4})\s*[년.\s\-`]*)?([1-4])\s*(?:분기|Q)
```
| 조각 | 읽는 법 |
|---|---|
| `(?: … )?` | 연도 부분은 **있어도 없어도 됨** (`2분기`처럼 연도 생략 대비) |
| `'?` | 작은따옴표 있거나 없거나 (`'24`) |
| `(\d{2,4})` | 연도 |
| `[년.\s\-`]*` | 연도-분기 사이 구분자: **`년`·점·공백·하이픈·백틱** 여러 개 |
| `([1-4])` | 분기 숫자 |
| `(?:분기|Q)` | '분기' 또는 'Q' |

→ **기수 걸러내기**: `제71기 2분기`에서 "71"이 연도로 잡히면 안 된다. 그래서 뽑은 연도가
**2015~2035 범위 밖이면 '연도 없음'으로 취급**한다(71 → 무시 → 연도는 축 ③이 채움).

> **왜 (b)를 (a)보다 먼저 하나?** (a)의 연도 부분은 "있어도 없어도 됨"이라 욕심이 많아서,
> `2Q24`를 (a)로 먼저 돌리면 "연도 없는 2분기"로 잘못 볼 수 있다. 연도가 확실히 뒤에 붙은
> (b)를 먼저 확인해 정확도를 높인다.

## 7. 축 ③ — 제출시점 휴리스틱 (12월 결산)

라벨이 없거나(`(-)`) 연도를 못 구하면, **"언제 제출했는가"**로 분기를 추정한다.
대부분이 12월 결산법인이고, 잠정실적은 분기 종료 직후에 낸다는 사실을 이용한다.

```
        제출월(md)                     추정 분기
   ┌──────────────────┐        ┌─────────────────────────┐
   │  1월 · 2월 · 3월   │  ───▶  │  전년(Y-1) 4분기 / 연간   │  (작년 결산 발표)
   │  4월 · 5월 · 6월   │  ───▶  │  당해(Y)   1분기          │
   │  7월 · 8월 · 9월   │  ───▶  │  당해(Y)   2분기          │
   │ 10월 ·11월 ·12월   │  ───▶  │  당해(Y)   3분기          │
   └──────────────────┘        └─────────────────────────┘
```

```python
def timing_quarter(y, mth):
    if mth <= 3: return (y-1, 4)   # 1~3월 제출 = 전년 4분기(연간)
    if mth <= 6: return (y, 1)
    if mth <= 9: return (y, 2)
    return (y, 3)                    # 10~12월 = 3분기
```

**연도만 모를 때(라벨에 분기는 있음)** 는 더 정밀하게 역산한다:

```python
def year_for_quarter(yd, md, q):
    # 분기 q의 종료월(q*3)이 제출월(md)보다 앞이면 올해, 아니면 작년
    return yd if q*3 <= md else yd-1
```

| 예 | 계산 | 결과 |
|---|---|---|
| 7월 제출, 2분기 | 2분기 종료월 6 ≤ 7 | 올해 |
| 7월 제출, 3분기 | 3분기 종료월 9 > 7 | 작년(아직 3분기 안 끝남) |
| 1월 제출, 4분기 | 4분기 종료월 12 > 1 | 작년 (작년 4분기 = 연간) |

### 7.1 결산월(회계연도 종료월) 검증 — 이 휴리스틱은 언제 안전한가?

이 휴리스틱은 "12월 결산"을 가정한다. 비(非)12월 결산법인이면 회계분기와 달력분기가 어긋날
수 있다. 그래서 **DART 정기보고서 제목으로 종목별 결산월을 실측**해 안전성을 검증했다.

- 소스: `xbrl_downloader/output/03_report_list.csv` 의 사업보고서 제목 `사업보고서 (2024.12)` →
  괄호 안 `연.월`이 결산월. 결산월을 바꾼 회사는 과거·현재 제목이 다르므로 **가장 최근 사업보고서**
  기준으로 채택. (추출: `src/extract_fiscal_months.py` → `data/fiscal_months.json`)
- 결과(대상 357종목): **356개 12월 · 1개 3월(SK오션플랜트 395400)**.
  - SK오션플랜트는 **12월→3월로 결산월을 변경**한 사례: `2022~2025=12월`, `2026=3월`.

**핵심 — 왜 이 데이터셋에선 12월 가정이 전부 안전한가:**

```
   회계분기 종료월이 "달력분기 경계(3·6·9·12월)"에 있으면
   ⇒ 회계 Q가 달력 Q와 어긋나도, "달력 기간(날짜)"은 동일하다.
   XBRL은 달력 날짜를 태깅하므로 결과가 같다.

   예) SK오션플랜트(3월 결산)의 회계 Q1 = 4~6월(달력 Q2)
       → 8월 제출 → timing=2분기 → 달력 4~6월  ✓ (라벨은 'Q1'이나 날짜는 동일)
```

우리 데이터의 결산월은 **12월·3월뿐이고 둘 다 경계월**이므로, 전 3,891건이 정확하다.
(부착된 `fiscal_month` 로 검증: **비경계 결산 종목 = 0**.) 결산월이 2·5·8·11월 등
비경계인 회사가 미래에 들어오면 이때만 별도 처리가 필요하다.

## 8. 전기·전년동기 파생 규칙

당기 (연도 Y, 분기 Q)가 정해지면 **전기·전년동기는 계산으로 결정**한다 —
회사가 라벨을 틀리게 써도 여기서 바로잡힌다.

```
                 당기 = Y년 Q분기
                     │
      ┌──────────────┼───────────────────┐
      ▼                                    ▼
   전기(previous)                      전년동기(yoy)
   = 직전 분기                          = 1년 전 같은 분기
   Q>1 → 같은 해 (Q-1)분기               = (Y-1)년 Q분기
   Q=1 → 전년(Y-1) 4분기
```

```python
def _derive(kind, y, u):          # 분기의 경우
    cur  = q_range(y, u)                              # 당기
    pq, py = (u-1, y) if u > 1 else (4, y-1)          # 전기: Q1이면 작년 Q4
    return cur, q_range(py, pq), q_range(y-1, u)      # 전년동기: 작년 같은 분기
```

**월·반기도 동일 원리**: 전월(월=1이면 작년 12월), 전년동월 / 직전반기, 전년동반기.

예) 당기 **2025 Q2** → 전기 **2025 Q1**, 전년동기 **2024 Q2**
예) 당기 **2025 Q1** → 전기 **2024 Q4**, 전년동기 **2024 Q1**

## 9. 축 ④ — 시계열 정합성 (카덴스 판정)

같은 **(종목 × 연결/별도)**의 공시들을 모아, 이 회사가 **분기로 내는지 월로 내는지**를 판정한다.
파라다이스(카지노매출)·현대차(판매대수)처럼 **월 단위**로 내는 회사는, 라벨이 없는(`(-)`) 건도
분기가 아니라 **월**로 해석해야 하기 때문이다.

```
 (종목, 기준) 시퀀스를 제출일순 정렬
        │
        ├─ 제출간격의 중앙값(med_gap) 계산
        │      · med_gap < 50일  →  "월간 필러"
        │      · 그 외            →  "분기 필러"
        │  (또는 라벨 종류가 '월'이 '분기'보다 많으면 월간)
        ▼
   월간 필러라면:
        저신뢰(low) 건들을 → 제출월−1 을 대상월로 하여 '월'로 재확정
        (잠정 월실적은 보통 익월 초에 제출되므로)
```

```python
gaps = [연속 제출일 간 일수…]
med_gap = 중앙값(gaps)
cadence = "month" if (월라벨 > 분기라벨) or (0 < med_gap < 50) else "quarter"
```

**예 — 파라다이스(연결)**: 매달 초 제출(간격 ≈ 30일) → med_gap ≈ 30 < 50 → **월간**.
라벨 없는 `2024-08-02` 공시 → 제출월(8)−1 = **2024년 7월**로 확정. ✔

## 15. 한계와 향후 개선

- **비(非)12월 결산법인** → **검증 완료(§7.1)**. DART 사업보고서 제목으로 결산월을 실측한 결과
  대상 357종목이 **12월·3월뿐이고 둘 다 달력분기 경계월**이라, 12월 가정 휴리스틱이 전 3,891건에서
  정확함을 확인했다. `data/fiscal_months.json`에 결산월 마스터를 두고 각 레코드에 `fiscal_month`를
  부착·검증(비경계 종목 0). **결산월이 2·5·8·11월 등 비경계인 회사**가 유입되면 그때만 별도 처리 필요.

- **개선 여지**: 비경계 결산월 회사 유입 시, `fiscal_month`로 회계분기의 달력범위를 직접 산출하는
